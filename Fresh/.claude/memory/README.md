# Project Memory

This folder stores only the memory needed for the current supervised UI workflow.

## Purpose

The memory system exists to support:

- UI planning
- iteration notes
- visual review checkpoints
- handoff summaries
- Queen One / ConnectUI standards

It is not a generic multi-agent memory framework.

## Kept folders

- `standards/`
  Queen One / ConnectUI standards and conventions
- `plans/`
  Task plans and scoped implementation notes
- `handoffs/`
  Review handoff notes and iteration summaries
- `session_checkpoints/`
  Resume-oriented checkpoints for active work
- `reports/`
  QA notes, visual review notes, and run summaries

## Working rule

When doing non-trivial UI work:

- use standards first
- write plans before build
- preserve key review notes
- preserve visual QA findings

This folder should stay small and workflow-specific.

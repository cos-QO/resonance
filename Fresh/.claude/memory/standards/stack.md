## Auto-Analysis Process

When `/prepare` runs, it will:
1. Inspect dependency manifests to identify runtime, frameworks, databases, and infrastructure
2. Detect build tools, package managers, and deployment targets
3. Map external services (auth, payments, storage, observability)
4. Populate the sections below

## Sections `/prepare` Will Populate

### Language & Runtime
- Primary language(s) and version constraints
- Runtime environment (Node, Python, JVM, .NET, Swift, etc.)

### Frameworks & Libraries
- Web framework, ORM, UI library, state management
- Major third-party dependencies

### Data Layer
- Database(s), caching layer, message queues, search

### Infrastructure & Deployment
- Hosting/cloud provider, containerization, CI/CD
- Monitoring, logging, error tracking

### Development Tooling
- Package manager, bundler, test runner, linter, formatter

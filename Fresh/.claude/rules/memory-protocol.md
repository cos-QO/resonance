# Memory Protocol

Use repo-local memory to preserve the parts of a UI task that are easy to lose across iterations.

## Always preserve

- the scoped plan
- key visual decisions
- deviations from the reference
- QA findings
- unresolved questions

## Preferred locations

- `plans/` for implementation plans
- `session_checkpoints/` for short resume notes
- `handoffs/` for review-ready iteration summaries
- `reports/` for QA and visual review notes

## Keep memory practical

- prefer short, high-signal notes
- do not turn memory into a generic archive
- keep artifacts aligned to real review and implementation needs

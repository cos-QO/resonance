# /training-kickoff — Full Training Lifecycle

Prepares infrastructure, generates scenarios, invokes `/train`, and updates the dashboard. Handles the full lifecycle so the user only types one command.

## Protected Targets (refuse immediately)

```
pm.md, skill-eval, hooks/*, settings.json, settings.local.json,
CLAUDE.md, version.json, training-protocol.md
```

If `$ARGUMENTS` matches any protected target, warn the user and stop.

## Phase 1: Interactive Discovery

### If `$ARGUMENTS` names a target
1. Resolve the target:
   - Check `/.claude/skills/$TARGET/SKILL.md` → skill
   - Check `/.claude/agents/$TARGET.md` → agent
   - Check `/.claude/rules/$TARGET.md` → rule
   - If not found: list available targets with `Glob("/.claude/agents/*.md")`, `Glob("/.claude/skills/*/SKILL.md")`, `Glob("/.claude/rules/*.md")` and report error
2. Confirm with user: "Train **{target}** ({type})? Default 10 iterations. Reply with a number to override, or press enter to start."
3. Parse iteration count from user reply (default 10)

### If `$ARGUMENTS` is empty or "discover"
Ask the user 3 questions (output as text, wait for replies):

**Question 1:**
> What would you like to train?
> - `agent` — improve an agent definition
> - `skill` — improve a skill's trigger accuracy & structure
> - `rule` — improve a rule file
> - `discover` — find and train the 3 weakest targets automatically

**Question 2** (skip if "discover"):
List available targets of the chosen type using Glob:
- Agents: `/.claude/agents/*.md` (mark protected ones with ⚠️)
- Skills: `/.claude/skills/*/SKILL.md` (mark `skill-eval` as protected)
- Rules: `/.claude/rules/*.md`

> Which target? (type the name)

**Question 3:**
> How many iterations? (default: 10, or 5 for discover mode)

### Discover mode resolution
If user chose "discover":
1. Run `/skill-eval all` via Skill tool to get scores for all skills
2. Present the 3 lowest-scoring targets
3. Ask user to confirm: "Train these 3? (y/n)"

## Phase 2: Infrastructure Setup

Check if `training/` exists at project root.

### If `training/` does NOT exist — create full structure:

1. Create directories:
   ```
   training/
   training/scenarios/
   training/data/
   training/metrics/
   ```

2. Create `training/.gitignore`:
   ```
   metrics/
   data/
   *.log
   ```

3. Create `training/README.md`:
   ```markdown
   # Training Infrastructure

   Generated by `/training-kickoff`. Contains scenarios, metrics, and the training dashboard.

   ## Structure
   - `scenarios/` — evaluation scenarios per target (JSON)
   - `data/` — training result logs (TSV)
   - `metrics/` — computed metrics and analysis
   - `index.html` — interactive training dashboard (open in browser)

   ## Usage
   ```
   /training-kickoff              # Full guided flow
   /training-kickoff developer    # Train a specific agent
   /training-kickoff discover     # Find and train weakest targets
   ```
   ```

4. Generate `training/index.html` — the training dashboard.
   Read the existing dashboard template from `training/index.html` if it exists. If not, generate a new one:
   - Use the standard dashboard structure (dark theme, Chart.js, AGENTS registry, EMBEDDED_TSV)
   - Initialize `EMBEDDED_TSV` with 16-column header row:
     ```
     timestamp\ttarget\titeration\tchange_description\tscore_before\tscore_after\tlines_before\tlines_after\tstatus\tcommit_hash\tchange_category\tfailure_reason\tfp_count\tfn_count\tscenario_file\tstandards_consulted
     ```
   - Include AGENTS registry entries for all agents found in `/.claude/agents/*.md`
   - Each agent entry needs: label, model, purpose, and 5 radar dimensions

### If `training/` exists — validate and patch:
1. Check each subdirectory exists (`scenarios/`, `data/`, `metrics/`), create missing ones
2. Verify `index.html` exists — if not, generate it as above
3. Leave existing files untouched

## Phase 3: Scenario Generation

Generate `training/scenarios/<target>-scenarios.json` for the resolved target.

### Read the target file first
Use Read tool to get full content of the target (agent/skill/rule `.md` file).

### For agents — generate:
```json
{
  "target": "<agent-name>",
  "type": "agent",
  "structural_quality": {
    "criteria": [
      {"name": "role_clarity", "description": "Role and boundaries clearly defined", "max_points": 10},
      {"name": "tool_access", "description": "Allowed tools match responsibilities", "max_points": 5},
      {"name": "workflow_steps", "description": "Clear step-by-step workflow", "max_points": 10},
      {"name": "output_format", "description": "Expected outputs specified", "max_points": 5},
      {"name": "disambiguation", "description": "NOT clauses prevent overlap", "max_points": 5},
      {"name": "model_fit", "description": "Model choice appropriate for task complexity", "max_points": 5},
      {"name": "conciseness", "description": "No redundant sections or repeated instructions", "max_points": 10}
    ],
    "max_total": 50
  },
  "trigger_scenarios": {
    "positive": ["<5 prompts that SHOULD route to this agent>"],
    "negative": ["<5 prompts that should NOT route to this agent>"]
  }
}
```

### For skills — generate:
```json
{
  "target": "<skill-name>",
  "type": "skill",
  "trigger_scenarios": {
    "positive": ["<5 prompts that SHOULD trigger this skill>"],
    "negative": ["<5 prompts that should NOT trigger this skill>"]
  }
}
```

### For rules — generate:
```json
{
  "target": "<rule-name>",
  "type": "rule",
  "effectiveness_criteria": [
    {"name": "specificity", "description": "Rules are concrete, not vague", "max_points": 15},
    {"name": "enforceability", "description": "Rules can be checked mechanically or by review", "max_points": 10},
    {"name": "coverage", "description": "Covers the domain without gaps", "max_points": 10},
    {"name": "clarity", "description": "Unambiguous language, no contradictions", "max_points": 10},
    {"name": "conciseness", "description": "No redundant rules or unnecessary prose", "max_points": 5}
  ],
  "max_total": 50
}
```

### Dashboard registration (agents only)
If the target is an agent and NOT already in the `AGENTS` registry in `training/index.html`:
1. Read the agent's `.md` file
2. Generate 5 radar chart dimensions relevant to the agent's role (e.g., for developer: code_quality, tool_usage, test_coverage, documentation, error_handling)
3. Add the entry to the AGENTS object in `index.html`

**Note**: The trainer agent auto-discovers scenario files generated here (`training/scenarios/<target>-scenarios.json`) during Step 2.5 of its loop. No explicit handoff is needed.

## Phase 4: Run Training

Invoke the training loop:

```
Skill("/train", args: "<target> --max-iterations <N>")
```

Where `<target>` is the resolved target name and `<N>` is the iteration count from Phase 1.

For discover mode, invoke `/train discover --max-iterations <N>`.

Wait for the Skill tool to complete. The trainer agent runs autonomously.

## Phase 4.5: Post-Training Validation (Optional)

After `/train` completes, optionally cross-validate the trainer's final score:

1. Run `/skill-eval <target>` via Skill tool (only for skills, skip for agents/rules)
2. Compare skill-eval's score with the trainer's final `score_after`
3. If discrepancy > 2 points: log a warning to the user:
   ```
   ⚠️ Score discrepancy: trainer reported {X}/50, skill-eval scored {Y}/50 (delta: {Z})
   Consider re-running training or investigating evaluation consistency.
   ```
4. If scores align (within 2 points): no action needed

Skip this phase if:
- Target is an agent or rule (skill-eval only scores skills)
- User explicitly requested fast mode or skipping validation
- Training resulted in no kept changes

## Phase 5: Dashboard Update & Cleanup

### 5a: Read trainer output
Check if `/.claude/memory/reports/training/results.tsv` exists.
- If it doesn't exist or is empty: report "No training results produced (target may already be optimal)" and skip to 5d.
- If it exists: read the full file content.

### 5b: Archive results
- Copy/append new rows to `training/data/results.tsv`
  - If file doesn't exist, create with TSV header row first
  - Append new data rows (skip header from source if present)

### 5c: Update dashboard EMBEDDED_TSV
1. Read `training/index.html`
2. Find the `EMBEDDED_TSV` constant (between backticks after `const EMBEDDED_TSV = \``)
3. Extract existing embedded rows
4. Merge with new rows from results.tsv
5. Dedup on composite key: `(timestamp, target, iteration)` — keep latest
6. Replace the EMBEDDED_TSV content in index.html using Edit tool
7. Write updated file

### 5d: Cleanup
- Delete `/.claude/memory/reports/training/results.tsv` (and the `training/` dir inside reports if empty)
  ```bash
  rm -f .claude/memory/reports/training/results.tsv
  rmdir .claude/memory/reports/training 2>/dev/null || true
  ```

### 5e: Report summary
Output to user:
```
## Training Complete

**Target**: {name} ({type})
**Iterations**: {kept}/{total} kept
**Score**: {before} → {after}
**Dashboard**: training/index.html (updated)

Open dashboard: `open training/index.html`
```

If multiple targets (discover mode), show a table.

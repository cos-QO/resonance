## Auto-Analysis Process

When `/prepare` runs, it will:
1. Detect the project's primary language(s) and frameworks
2. Inspect dependency manifests (`package.json`, `requirements.txt`, `Cargo.toml`, `Package.swift`, etc.)
3. Scan existing code for naming and style conventions
4. Populate the sections below with project-specific standards
5. Update this file with findings and timestamp

## Sections `/prepare` Will Populate

### Project Structure
- Project root, type, repository info
- Primary language(s) and configuration format

### Naming Conventions
- File, variable, function, class, constant naming rules

### Code Quality Standards
- Language-specific linting rules (PEP 8, ESLint config, etc.)
- Type-checking, docstring/comment requirements, formatter

### Testing Standards
- Test framework(s), file patterns, coverage targets

### Version Control Standards
- Primary branch, commit format, branch naming

### Development Workflow
- Code review, linting, testing, documentation gates

## Auto-Analysis Process

When `/prepare` runs, it will:
1. Detect the project's formatter and linter configurations
2. Inspect existing code for style patterns (indentation, line length, quote style, etc.)
3. Document project-specific stylistic rules
4. Populate the sections below

## Sections `/prepare` Will Populate

### Formatting Rules
- Indentation (spaces vs tabs, width)
- Line length limit
- Quote style, trailing commas, semicolons (language-dependent)

### Formatter & Linter
- Configured formatter (Prettier, Black, rustfmt, swift-format, etc.)
- Linter rules (ESLint, Ruff, Clippy, SwiftLint, etc.)

### Language-Specific Patterns
- Error handling idioms
- Module/import organization
- Type usage (when strict typing is enforced)

## Auto-Analysis Process
When `/prepare` runs, it will:
1. Scan for CSS files, design tokens, component libraries

## Design System Components to Analyze

### Color System
- Primary colors and variants
- Secondary color palette  
- Semantic colors (success, error, warning, info)
- Neutral grays and backgrounds
- Color naming conventions

### Typography
- Font families and weights
- Text scales and sizes
- Line heights and spacing
- Typography hierarchy
- Text utilities and classes

### Spacing & Layout
- Spacing scale (margins, padding)
- Grid system and breakpoints
- Layout patterns and utilities
- Container sizes and constraints

### Component Patterns
- Button variants and states
- Form elements and validation
- Navigation patterns
- Card and content layouts
- Modal and overlay patterns

### Design Tokens
- CSS custom properties / variables
- Theme structure and organization
- Token naming conventions
- Dark/light mode support

## UI Framework Detection
- Component library used (Material-UI, Ant Design, Chakra UI, etc.)
- CSS framework (Tailwind, Bootstrap, Bulma, etc.)
- Styling approach (CSS-in-JS, SASS, PostCSS, etc.)
- Design tool integration (Figma tokens, etc.)
